# 提案：用户权限管理系统 (User Permission Management)

## 1. 背景与目标
当前系统仅有简单的 `admin/user` 角色区分，无法满足细粒度的资源访问控制需求。
本提案旨在实现针对核心资源（智能体、数据集、接口）的用户级权限管理，并为未来基于角色的访问控制（RBAC）预留架构支持。

## 2. 核心变更
### 2.1 资源权限模型
引入通用的资源权限表，支持对以下资源的授权：
- **智能体 (Agents)**: 控制用户能通过 ChatBI 或 API 访问哪些 Agent。
- **数据集 (Datasets)**: 控制用户能检索或管理哪些知识库。
- **对外接口 (External APIs)**: 控制用户是否有权通过 `v1` 路径调用平台能力（如 Chat, Search 等）。内部 Portal 接口无需此授权。

### 2.2 角色 (预留)
虽然本次主要实现直接的用户-资源授权，但架构设计将包含 `Role` 概念，以支持未来“角色包含一组权限，用户关联角色”的标准 RBAC 模式。

### 2.3 用户管理界面
在前端用户管理模块增加权限配置功能：
- 支持多 Tab 视图切换不同资源类型。
- 支持资源的批量选择与授权。

### 2.4 性能与缓存
- 权限数据加载至 Redis 缓存。
- 权限变更时自动刷新缓存，确保实时性。

## 3. 交付物
- 数据库变更脚本 (SQL)。
- 后端权限管理 API 及缓存逻辑。
- 前端用户权限设置 UI。
- 权限校验中间件/依赖。
