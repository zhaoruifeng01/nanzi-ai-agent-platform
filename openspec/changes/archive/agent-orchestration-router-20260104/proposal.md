# Proposal: Agent Orchestration Router

## Why
目前系统支持多个智能体，但用户需要手动选择特定的智能体进行对话。随着智能体数量的增加（例如：专攻运维的 Agent、专攻财务数据的 Agent、通用知识 Agent），用户很难判断哪个智能体最适合当前问题。
实现“自动编排（Orchestration）”可以提供一个统一的入口，让系统自动识别用户意图并调度最合适的专家智能体，极大提升用户体验和系统的智能化程度。

## What Changes
1. **意图调度器 (Agent Router)**：引入一个中心化的调度逻辑，能够基于语义相似度（Embedding）或 LLM 推断，将用户请求分发至最匹配的智能体。
2. **智能体元数据增强**：在 `AIAgent` 模型中增加 `description` 或 `capabilities` 字段，用于描述智能体擅长的领域，作为路由的依据。
3. **编排模式 (Orchestration Mode)**：在聊天接口中增加一种特殊的模式，支持“Master Agent”调用多个“Sub-Agents”并整合结果。
4. **管理后台适配**：在智能体管理界面增加“功能描述”和“路由权重”等配置项。

## Goals
- 提供统一对话入口，无需手动切换智能体即可处理复杂跨域问题。
- 实现智能体之间的协同，支持 A 智能体调用 B 智能体。
- 保证系统响应速度，优先采用语义路由降低 Token 成本。
