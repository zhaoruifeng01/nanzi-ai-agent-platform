# ChatBI Few-Shot 注入策略优化记录 (2026-03-25)

针对用户反馈经验库 (ChatBI Examples) 在 SQL 生成中的指导作用，本次完成了一波深度优化，旨在解决“Lost-in-Middle”效应（长 Prompt 淹没核心案例）以及高相似案例指导强度不足的问题。

## 📍 核心改动点汇总

### 1. 注入位置优化 (`data_executor.py`)
- **注入点 1 (System Prompt 头部)**: 将检索到的 Few-Shot 案例块从 System Prompt 的**末尾**移动到了**最开始**。确保模型在处理复杂的 ReAct 指令和表结构之前，先建立正确的参考锚点。
- **注入点 2 (Schema 获取后的二次提醒)**: 在 `get_dataset_schema` 成功返回表结构后，向对话序列中额外注入一条简短的 `SystemMessage`。这条消息旨在模型刚看完长篇大论的实时表结构、准备写 SQL 的**关键时刻**，重新唤醒它对历史案例核心逻辑的记忆。

### 2. 引导提示词精细化 (`chatbi_example_service.py`)
- **阈值下调 (0.80 → 0.65)**: 将“强制参考”案例的置信度要求从 `0.80` 降低到 `0.65`。满足实际环境中相似度难以破 0.8 但仍具有极高参考价值的场景。
- **强规则三层架构**: 在 Prompt 引导中划分了明确的复用边界：
  - ✅ **必须复用**: 物理表名 (FROM/JOIN)、多表连接关系、核心聚合函数/算子。
  - 🔧 **允许适配**: WHERE 时间窗口、过滤数值、SELECT 字段别名（按当前问题对齐）。
  - ❌ **禁止改动**: 除非当前库里确实没这表，否则绝对禁止“幻觉”捏造。
- **语义化标签**: 移除了机器视角的 `(相似度: 0.72)`，替换为 `【极高匹配】`、`【较高匹配】` 等模型更容易理解的语义标签；并在最高相似度案例前加 `⭐最优先参考` 标记。

### 3. 构建案例 Reminder (`chatbi_example_service.py`)
- 新增 `build_few_shot_reminder()` 方法，用于生成注入点 2 的简短提醒。内容精简展示：案例核心表名、SQL 预览（前3行逻辑线索），不重复整段 SQL 以节省 Context 窗口空间。

## 🚀 预期影响与验证建议

### 预期效果
- **减少“想当然”选表**: 即使实时 Schema 很大，模型也会优先对照经验库中的表名。
- **减少 ReAct 迭代次数**: 由于注入了经过验证的 JOIN 关系，模型“盲目试错”生成的频率会显著下降。
- **复杂计算复用性增强**: 针对环比、同比、特定业务口径的 SQL 结构，复用率会更高。
