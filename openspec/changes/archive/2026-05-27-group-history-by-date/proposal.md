## 为什么需要此变更 (Why)

目前，云枢智能体平台嵌入式聊天（EmbedChat）的历史会话侧边栏中，历史记录是以单个 Trace 进行完全平铺展示的。这种平铺不仅导致界面随着测试或日常对话的增多而显得极其冗长无序，更使得开发人员和用户难以根据时间跨度快速回溯特定的历史对话。同时，平台缺乏“批量清理”机制，导致开发测试过程中产生的大量临时、冗余垃圾会话只能逐个手动点击删除，极其影响工作效率与用户体验。

因此，亟需引入历史会话按日期分组显示及分组一键批量删除的功能，以提供更符合人类心智模型的结构化回溯视图和高效的垃圾数据清理手段。

## 变更内容 (What Changes)

- **会话按日期智能分组**：侧边栏会话历史列表将自动按照“今天”、“昨天”、“3天前”、“7天前”以及“更早”这五个直观的时间区间进行逻辑分拣与分组展示。
- **分组一键批量删除**：在各个日期分组的 Header（标题行）上，新增一个醒目的“一键删除”图标按钮。点击确认后，能够批量删除该组内的所有会话，极大提高清理效率。
- **批量删除后端接口**：后端新增专门的批量删除 API `/api/v1/chat/history/batch-delete`，通过接受一组成员 UUID 执行原子化删除，减少多次网络往返，保护系统性能。

## 能力范围 (Capabilities)

### 新增能力 (New Capabilities)
- `group-history-by-date`: 覆盖在云枢平台嵌入式聊天（EmbedChat）侧边栏中，将会话历史按相对日期分组展示、前端分组 Header 批量删除动作触发、以及后端 `/api/v1/chat/history/batch-delete` 批量物理/逻辑删除的端到端能力。

### 修改能力 (Modified Capabilities)
- 无

## 涉及范围与影响 (Impact)

- **前端交互与组件**：
  - `ChatHistorySidebar.vue`：UI 结构由平铺渲染重构为双重循环嵌套；添加分组 Header 组件及批量删除确认动作。
  - `EmbedChat.vue`：引入 `groupedHistoryList` 聚合分组计算逻辑；对接子组件抛出的 `@delete-group` 批量删除事件，调用接口并重新刷新列表。
- **后端服务与接口**：
  - 新增批量删除接口，接受 `conversation_ids` 数组，对相关的会话记录（如 Trace 主表、审计日志及消息历史记录等）进行统一事务内清理。
