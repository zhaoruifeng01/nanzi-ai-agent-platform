# 智能体技能管理（Skills Management）模块开发提案

## 1. 背景与问题陈述
目前云枢智能体中台已经打通了原子工具链（Tools）的配置。但是在大型企业生产环境中，单一行动的原子工具无法有效承载**高阶行为纪律**和**专家级战术工作流**。

智能体不仅需要手脚（原子工具），更需要大脑和经验（Skills，技能体系）。目前宿主机的 `~/.agent/skills` 目录（容器内 `/app/data/skills`）下已经存放了多个高阶技能。为了能够让管理员与开发者彻底掌控智能体的“认知库”，我们不仅需要能够可视化查阅技能，**更需要支持在线创建、上传辅助脚本、编辑规则描述、以及彻底删除不再使用的技能**，打通技能全生命周期的可视化管理。

另外，为了帮助系统管理员更清晰地了解“如何从外部生态安装高品质 Skills”，我们需要在页面提供**直观的帮助指引（如 “？” 悬停气泡或弹窗说明）**，引导用户如何使用官方 `Skills CLI` 进行下载安装，并提供链接至开放生态市场。

---

## 2. 方案目标
在“智能体开发平台”的菜单下，新增 **「技能管理」 (Skills Management)** 子模块，全面支持：
1. **列表查阅与过滤**：动态扫描并读取解析各个技能目录中的 `SKILL.md`，提供磨砂玻璃卡片式的“技能市场”看板。
2. **在线创建技能**：支持直接输入 Skill ID（文件夹名）、技能名称与描述，自动创建物理目录并初始化 `SKILL.md` 模板。
3. **在线编辑与文件管理**：
   - 可以在抽屉详情中，直接编辑 `SKILL.md` 规则并保存。
   - 支持向特定技能子目录下上传新的脚本（如 Python/JS 辅助脚本），或直接删除其内个别文件。
4. **彻底删除技能**：支持一键清空并删除整个技能目录。
5. **帮助指引中心**：在页面头部提供 **「？」帮助指引弹窗**，说明如何使用 `npx skills add <repo> --skill <name>` 将生态技能安装至平台的本地映射路径 `./data/skills` 下，并提供外链指向 [skills.sh](https://www.skills.sh/) 开放市场。
6. **极客详情抽屉**：双栏布局，左半区配备支持直接编辑保存的 Markdown 规则编辑器，右半区配备文件树及文件上传/删除动作列。

---

## 3. 功能范围 (Scope)

### ✅ 包含功能 (In Scope)
* **后端 API**：
  - `GET /api/portal/skills`：遍历 `/app/data/skills`，动态读取解析 `SKILL.md` 的 YAML 头部，返回列表。
  - `POST /api/portal/skills`：新建技能。创建专属文件夹并初始化默认 `SKILL.md` 结构。
  - `GET /api/portal/skills/{skill_id}`：获取特定技能详情，含 `SKILL.md` 完整 Markdown 内容和文件目录树结构。
  - `DELETE /api/portal/skills/{skill_id}`：删除指定技能目录（安全校验，防止 rmtree 越界）。
  - `PUT /api/portal/skills/{skill_id}/files`：在线保存或编辑技能目录下的指定文本文件（如 `SKILL.md` 或脚本代码）。
  - `POST /api/portal/skills/{skill_id}/upload`：上传新的文件到技能指定目录（支持 multipart 上传）。
  - `DELETE /api/portal/skills/{skill_id}/files`：删除技能目录下的单个或多个指定文件。
* **前端 UI**：
  - **侧边栏集成**：在“智能体管理”下方无缝挂载“技能管理”导航。
  - **帮助引导**：头部明显的 **「？」帮助按钮**，弹出精美说明卡片，指导使用 `npx skills add` CLI 工具，内嵌 [skills.sh](https://www.skills.sh/) 链接。
  - **技能网格看板**：响应式的卡片列表展示，含搜索过滤、新建技能入口、删除技能入口。
  - **文件树与规则详情**：以 Drawer 抽屉形式，左侧配备编辑器编辑规范，右侧展现文件树并提供“上传文件”、“删除文件”功能。
* **容器映射与防穿越保护**：
  - 增、删、改、查等所有文件系统操作，必须实施绝对的防路径穿越安全校验，拦截一切非法越权操作。
  - 所有 API 及前端导航访问受到 `menu:skills_management` 统一权限卫士保护。

### ❌ 不包含功能 (Out of Scope)
* **技能版本控制（Git History）**：暂不支持在线查看技能文件的 Git 历史记录与提交回滚，以物理文件的增删改查为主。
