# Proposal: 增强智能体日志与可视化 (Enhance Agent Logging)

## 背景 (Background)
目前的智能体调试功能比较有限，虽然界面上会流式显示“正在思考”、“正在查询”，但缺乏详细的执行链路信息（如：具体调用了哪个工具？传了什么 SQL？数据库返回了什么 JSON？）。这让开发者和用户在遇到“查不到数据”或“回答错误”时难以排查根因。

## 目标 (Goal)
实现一套完整的智能体执行链路日志系统，并在聊天界面提供直观的“查看日志”入口，允许用户回溯每一次对话的完整技术细节。同时，完善审计日志，确保这些关键的执行步骤也被持久化记录。

## 范围 (Scope)
- **后端 (Backend)**:
  - 改造 `AgentService`，使其在执行过程中收集结构化的 Trace 日志（Step, Tool Name, Input, Output, Status）。
  - 将这些 Trace 日志与当前的 `trace_id` 关联。
  - 增强审计日志（Audit Log），将智能体的执行摘要或详情也作为审计的一部分进行存储（存入新的 `ai_agent_execution_traces` 表）。
- **前端 (Frontend)**:
  - 在聊天气泡旁增加“📜 查看日志”按钮。
  - 实现一个侧边抽屉（Drawer）或弹窗，以时间轴形式展示执行步骤。
  - 支持 JSON 格式化高亮显示，方便查看复杂的 SQL 和数据结果。

## 计划 (Timeline)
- **Phase 1**: 后端日志结构设计与采集实现。
- **Phase 2**: 数据库存储与审计关联。
- **Phase 3**: 前端可视化组件开发。
