## 新增需求

### 需求: 智能体任务管理工具
系统必须提供内置工具，允许智能体代表用户管理定时任务。

#### 场景: 通过自然语言创建任务
- **当** 用户说“帮我设定每天早上 8 点查 PUE”时
- **并且** 智能体调用 `create_recurring_task(name="PUE巡检", cron="0 8 * * *", prompt="查询 PUE")`
- **那么** 系统在数据库中成功创建任务

#### 场景: 查询我的任务
- **当** 用户问“我有哪些定时任务？”时
- **并且** 智能体调用 `get_my_tasks()`
- **那么** 系统返回该用户所有的活跃任务列表

### 需求: 工具权限范围
这些管理工具必须仅能操作当前用户上下文拥有的任务。

#### 场景: 防止跨用户访问
- **当** 用户 A 要求删除属于用户 B 的任务时
- **并且** 智能体尝试调用 `delete_task(用户B的任务ID)`
- **那么** 工具执行必须失败并提示“权限拒绝”
