## 新增需求

### 需求: 分布式任务调度
系统必须使用分布式调度机制，确保在多个服务实例运行的情况下，定时任务在指定时间仅被触发一次。

#### 场景: 集群环境下的单次执行
- **当** 多个服务实例（如 3 个节点）同时运行时
- **并且** 一个定时任务（如 08:00 的日报）到达触发点
- **那么** 只有一个实例能成功抢占执行锁
- **并且** 任务仅被执行一次
- **并且** 其他实例在该周期内跳过执行

### 需求: 支持 Cron 表达式
系统必须支持标准的 Cron 表达式来定义任务周期，并提供语法校验。

#### 场景: 有效的 Cron 输入
- **当** 用户创建一个 cron 为 `0 8 * * *`（每日早 8 点）的任务时
- **那么** 系统接受输入并正确计算下一次运行时间

#### 场景: 无效的 Cron 输入
- **当** 用户输入错误的 cron `99 99 * * *` 时
- **那么** 系统拒绝创建请求并返回明确的错误提示

### 需求: 用户上下文恢复 (身份模拟)
系统在后台执行定时任务时，必须能够恢复创建者的用户上下文（身份、权限）。

#### 场景: 防止越权访问数据
- **当** 一个定时任务尝试查询受限数据集时
- **并且** 任务创建者没有该数据集的访问权限
- **那么** 执行必须失败并返回“权限拒绝”错误
- **并且** 系统记录该未授权访问尝试

### 需求: Prompt 变量替换
系统必须支持在任务指令中使用动态时间变量，并在执行时将其替换为实际日期字符串。

#### 场景: 动态日期替换
- **当** 指令为“查询 {{yesterday}} 的数据”的任务在 2023-10-02 执行时
- **那么** 系统发给智能体的最终指令应为“查询 2023-10-01 的数据”

### 需求: 执行结果通知
系统应支持将任务执行结果推送到配置的外部渠道（Webhook）或内部历史日志中。

#### 场景: Webhook 通知
- **当** 任务成功执行完成时
- **并且** 任务配置了 Webhook URL
- **那么** 系统将智能体的响应内容发送到该 Webhook
