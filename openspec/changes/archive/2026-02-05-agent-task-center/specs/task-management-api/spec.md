## 新增需求

### 需求: 任务 CRUD 操作
API 必须提供创建、读取、更新和删除定时任务的端点。

#### 场景: 列出用户任务
- **当** 已认证用户请求 `GET /api/v1/tasks` 时
- **那么** 系统返回该用户创建的所有任务列表
- **并且** 响应中排除敏感字段（如加密的上下文）

#### 场景: 更新任务周期
- **当** 用户请求 `PATCH /api/v1/tasks/{id}` 并传入 `{"cron": "0 9 * * *"}` 时
- **那么** 系统更新任务周期
- **并且** 调度器立即重新计算下一次运行时间

### 需求: 手动触发执行
API 必须提供一个端点，允许手动触发定时任务以进行测试或临时执行。

#### 场景: 立即执行
- **当** 用户请求 `POST /api/v1/tasks/{id}/run` 时
- **那么** 系统立即异步调度该任务的执行
- **并且** 返回用于跟踪状态的 `trace_id`

### 需求: 查询执行历史
API 必须提供查询任务执行日志的端点。

#### 场景: 查看任务日志
- **当** 用户请求 `GET /api/v1/tasks/{id}/logs` 时
- **那么** 系统返回该任务过去执行的分页列表，包含状态、耗时和结果
