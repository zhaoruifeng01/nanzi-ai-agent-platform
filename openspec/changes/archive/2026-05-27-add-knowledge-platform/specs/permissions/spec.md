## ADDED Requirements

### Requirement: 知识库开发平台权限

系统 SHALL 将知识库开发平台的菜单和操作纳入现有权限配置体系，并在前后端同时执行权限控制。

#### Scenario: 配置知识库菜单权限

- **WHEN** 管理员在角色管理中配置菜单权限
- **THEN** 系统提供「知识库开发平台」「知识库管理」「检索测试」相关菜单权限选项
- **AND** 保存后拥有对应权限的用户可以看到相应菜单入口

#### Scenario: 配置知识库操作权限

- **WHEN** 管理员在角色管理中配置操作权限
- **THEN** 系统提供创建知识库、删除知识库、上传文档、删除文档、解析文档和执行检索测试的操作权限选项
- **AND** 保存后前端根据权限展示或隐藏对应操作按钮

#### Scenario: 后端强制校验知识库操作权限

- **WHEN** 用户直接调用知识库创建、删除、上传、文档删除、文档解析或检索测试 Portal API
- **THEN** 后端必须校验用户是否拥有对应操作权限
- **AND** 无权限用户请求被拒绝并返回明确的无权限错误

#### Scenario: 有操作权限的用户执行知识库管理

- **WHEN** 用户拥有创建、删除、上传、文档删除、文档解析或检索测试中的任一操作权限
- **THEN** 系统允许用户执行对应操作
- **AND** 该能力不限定为管理员角色

#### Scenario: 非管理员知识库列表过滤

- **WHEN** 非管理员用户请求知识库列表或打开知识库选择器
- **THEN** 系统按照该用户已有资源权限过滤可见知识库
- **AND** 用户不可见未授权的 RAGFlow 知识库
