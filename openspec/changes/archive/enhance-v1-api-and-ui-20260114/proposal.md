# Proposal: V1 API 权限管控与 UI 体验增强

## Background
当前平台已具备基础的权限模型（用户-资源），但仅覆盖了智能体和数据集，缺少对对外开放 API (V1) 的精细化管控。同时，前端智能体管理界面的交互体验（开关状态、预览方式）需要现代化改进，以提升运维效率。

## Goal
1.  **API 权限闭环**：将 V1 接口纳入权限管理体系，支持动态发现接口并分配给用户/角色，并在调用时强制鉴权。
2.  **用户画像接口**：提供标准化的用户详情查询接口，便于第三方系统集成。
3.  **交互体验升级**：采用 macOS 风格的开关控件，提供便捷的 EmbedChat 预览入口。

## Scope
- **Backend**:
  - 扩展权限服务，支持 `api` 类型资源。
  - 实现 V1 接口的动态扫描与注册。
  - 新增 `GET /api/v1/users/profile` 接口。
  - 在 V1 API 依赖层增加权限校验逻辑。
- **Frontend**:
  - 用户/角色权限管理界面增加“接口权限” Tab。
  - 智能体管理界面：状态栏改为 Toggle Switch，操作栏增加“预览”按钮。
  - 确认 EmbedChat 支持 URL 参数鉴权（已支持）。

## Changes
- `permissions`: 新增 API 资源类型定义及校验逻辑。
- `v1-chat`: 新增用户画像接口规范。
- `agent-management`: UI 交互规范更新。
