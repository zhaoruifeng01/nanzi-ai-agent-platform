# 登录认证功能增强

## 背景 (Why)
当前系统仅支持 API Key 登录。为了提供更符合用户习惯的体验，我们需要引入账号密码登录方式。
同时，为未来集成 "YES 系统" 做准备，并实施更严格的安全策略（如：用户自行设置密码，管理员不可见用户密码）。

## 变更内容 (What Changes)

### 认证方式 (Authentication Methods)
-   **账号密码登录 (新增/默认):** 用户使用用户名和密码登录。
-   **API Key 登录 (现有):** 保留作为备选登录方式。
-   **YES 系统集成 (预留):** 增加一个预留的登录选项卡。

### 用户安全流程 (User Security Flow)
-   **数据库变更:** `ai_agent_users` 表新增 `password_hash` 字段。
-   **初始化设置:**
    -   现有用户（或管理员新创建的用户）初始状态下没有密码。
    -   如果用户尝试用密码登录但未设置密码，系统将提示错误：“请先使用 API Key 登录后设置密码”。
-   **密码管理:**
    -   **管理员限制:** 管理员 **不能** 设置或修改其他用户的密码（遵循安全最佳实践）。
    -   **用户自服务:** 用户可以在 "个人中心"（Dashboard 左下角）设置或修改自己的密码。
    -   **首次设置:** 用户首次设置密码时，必须先通过 API Key 登录系统。

### 界面更新 (UI Updates)
-   **登录页面:** 改为选项卡式布局 (账号密码 | API Key | YES 系统)。
-   **侧边栏/用户菜单:** 新增 "个人中心" 入口，包含 "修改密码" 功能。

## 用户影响 (User Impact)
-   用户将看到全新的登录界面。
-   首次登录的用户需要遵循 "API Key 登录 -> 设置密码" 的流程。
