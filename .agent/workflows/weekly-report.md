# 周报生成工作流

## 概述
用于生成云枢智能体平台开发周报的工作流，支持生成两个版本：
- **GitLab版本**：详细技术周报，包含具体实现细节
- **邮件版本**：简洁商务周报，适合邮件发送

## 使用方法

### 1. 生成周报
运行以下命令读取本周git log并生成周报：
```bash
git log --since="1 week ago" --pretty=format:"%h - %ad - %s" --date=short --author="chenxiaolong"
```

## 生成步骤

### 步骤1：数据收集
```bash
# 获取本周提交记录
git log --since="1 week ago" --oneline --author="chenxiaolong"

# 获取详细提交信息
git log --since="1 week ago" --pretty=format:"%h - %ad - %s" --date=short --author="chenxiaolong"
```

### 步骤2：内容分类
根据提交信息将工作内容分类为：
- 🎯 核心功能开发
- 🔒 系统安全加固  
- 🎨 UI/UX体验提升
- ⚡ 性能与稳定性优化
- 🔧 技术架构改进
- 🧪 测试与质量保证

### 步骤3：生成两个版本
- **邮件版本**：简洁明了，突出主要成果
- **GitLab版本**：详细技术细节，便于技术回顾

## 注意事项

1. **时间范围**：默认为最近一周，可根据需要调整
2. **作者过滤**：只统计指定作者的提交
3. **内容准确性**：确保技术描述准确无误
4. **格式统一**：保持markdown格式规范

## 自动化脚本（可选）

可以创建自动化脚本来简化周报生成过程：

```bash
#!/bin/bash
# weekly-report-generator.sh

echo "=== 云枢智能体平台开发周报生成 ==="
echo "请选择周报版本："
echo "1) 邮件版本"
echo "2) GitLab版本"
echo "3) 两个版本都生成"

read -p "请输入选择 (1/2/3): " choice

case $choice in
    1)
        echo "生成邮件版本..."
        # 调用邮件版本模板
        ;;
    2)
        echo "生成GitLab版本..."
        # 调用GitLab版本模板
        ;;
    3)
        echo "生成两个版本..."
        # 生成两个版本
        ;;
    *)
        echo "无效选择"
        ;;
esac
```
