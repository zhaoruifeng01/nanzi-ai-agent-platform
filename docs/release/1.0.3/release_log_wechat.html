🚀 南孜 AI 智能体平台 · v1.0.3 重磅发布

数据门户体验升级 · 查数可靠性全面强化

GitHub Repository： RandyChen1985/nanzi-ai-agent-platform

v1.0.3 版本是一次数据门户体验与查数可靠性升级。在本次更新中，平台全新推出「我的数据门户」个性化导航与场景推荐能力，并上线 /dataset_menu 系统级数据能力导航；同时借鉴 OpenClaw 架构引入会话串行、工具循环熔断与上下文压缩等运行时增强，大幅强化 ChatBI 物理表防猜表与数据集归属强一致性校验。

本次变更合并了自 v1.0.2（从 2b6be3b 至今）共 25 个提交，涉及 64 个文件、约 9,000 行新增代码。

🚀 Key Features

01 🏠 数据门户（Data Portal）全面体验升级

● 「我的数据门户」个性化导航：记录用户点击偏好，按使用习惯排序展示数据集入口；全面美化 UI/UX，解决气泡复制与刷新按钮重合冲突。

● 场景卡片推荐问题换一批：支持场景卡片推荐问题实时刷新，并优化兜底条数。

● 快捷提问隐式携带物理表名：数据门户快捷提问自动附带物理表名上下文，提升首轮查数命中率。

● 超时降级静默自愈刷新：数据门户超时后自动静默刷新恢复；刷新成功 Toast 提示，加载态温馨提示轮播。

02 🧭 /dataset_menu 系统级数据能力导航

● 系统指令入口：新增 /dataset_menu，由 LLM 按数据集生成中文导航与 quick 追问按钮。

● 引用块 + 表格样式：以「表名 / 业务范围 / 示例问题」三列表格展示，示例问题内嵌可点击 quick 按钮；明确展示「系统 · 数据导航」身份。

03 🛡️ ChatBI 查数安全与准确性强化

● 物理表防猜表与强一致性校验：防止模型将业务术语、指标块或中文表名误当作物理表名执行 SQL。

● Schema 检索输出格式统一：对齐向量检索、MySQL 兜底、RAGFlow 与 HTTP API；工具日志显示命中条数与 token 估算。

● 业务友好拦截文案：将技术术语改为业务友好提示；约束 quick 追问区块必须位于图表之后。

04 ⚙️ OpenClaw 借鉴：会话运行时增强

● 会话串行（Session Lane）：防止同一会话并发请求互相干扰，避免状态污染。

● 工具循环熔断：新增 ping-pong / 全局熔断检测，防止工具调用陷入死循环。

● 上下文压缩与 followup 队列：超长历史注入确定性摘录；受控重试与内容撤回改善连续追问体验。

05 🎯 意图识别与路由优化

● 问候场景短路路由：纯寒暄直接走通用助手并跳过分类 LLM，显著降低响应延迟。

● 联网搜索与知识库区分：实时信息走通用助手，防止未绑定知识库时误走知识库路径失败。

● 问候误拦截修复：反幻觉门控按意图触发，避免「你好」被误杀；Schema 弱命中友好兜底。

06 🎨 管理界面与前端体验优化

● 优化 Agent 管理、元数据管理、系统参数配置等界面交互与提示；修复记忆管理中心 FT.INFO NaN 序列化报错。

07 📄 开源许可

● 项目正式添加 MIT License 开源许可证。

⚠️ Breaking Changes & Migration Notes

从 v1.0.2 升级至 v1.0.3 时，请特别注意以下变更：

项目 说明
数据库变更 须执行 V78-chatbi_quick_suggestions_placement.sql，同步 ChatBI quick 按钮排版约束。
元数据重新同步 _metrics.txt 新增注释声明；建议对受影响数据集重新同步元数据。
会话串行行为 同一会话并发请求将串行执行，高频并发场景下响应可能略有排队。
查数拦截更严格 使用业务术语或中文表名直接写 SQL 将被拦截并引导回 Schema 检索，属预期行为。

🗄️ Database Incremental Upgrades

从 v1.0.2 升级至 v1.0.3 期间，平台数据库共引入了 1 个增量 SQL 升级脚本：

脚本文件 核心变更内容
V78-chatbi_quick_suggestions_placement.sql 更新 ChatBI V8 提示词，约束 quick 追问区块必须位于全文最末尾（图表之后），并补充自检清单项。

📦 Upgrade Guide

从 v1.0.2 升级

# 1. 拉取最新代码

git fetch origin && git checkout dev && git pull origin dev

# 2. 更新依赖

source venv/bin/activate

pip install -r requirements.txt

# 3. 执行数据库迁移

./db-prod/apply-sql-native.sh

# 4. 重新编译前端并启动

cd frontend && npm install && npm run build && cd ..

./dev.sh

完整部署说明请参考 HOW_TO_INSTALL.md。
