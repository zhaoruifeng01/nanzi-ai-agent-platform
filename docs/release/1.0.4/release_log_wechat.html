🚀 南孜 AI 智能体平台 · v1.0.4 重磅发布

联邦查数 · 数据门户 2.0 · ChatBI 运行时全面进化

GitHub Repository： RandyChen1985/yunshu-ai-agent-platform

v1.0.4 版本是一次跨数据集联邦查数、数据门户 2.0 与 ChatBI 运行时全面进化的大版本更新。平台全新上线联邦查询引擎与链路追踪 Trace Span 画布；数据门户升级为「一数据集一场景」并支持黄金 SQL 报表暂存；ChatBI 引入 SQL 安全沙箱、空结果自动诊断重试与模块化架构重构；EmbedChat 新增 Mermaid / HTML / PDF / 图片 Canvas 在线预览。

本次变更范围自 528d43c（含）至 3534d54，共 104 个提交，涉及 255 个文件、约 33,000 行新增代码。

🚀 Key Features

01 🔗 跨数据集联邦查询引擎

● 联邦执行与 synthesis：跨数据集 SQL 计划分解、逐节点 repair 与中文表头汇总输出。

● 自适应升级：单源跨表引用自动引导升级联邦；子查询误判升级为整计划 repair。

● Trace Span 画布：链路追踪支持嵌套步骤父子层级展示与计时审计。

02 🏠 数据门户 2.0

● 一数据集一场景：按业务场景合并卡片；「我常问」管理、多数据集折叠与卡片置顶。

● /dataset_portal：门户主指令升级，支持钉住、缓存失效、换一批 4 秒冷却与表级推荐提问。

● 移动端全面优化：抽屉、顶栏滚动、快捷指令收进顶部栏。

03 🛡️ ChatBI SQL 安全沙箱与智能修复

● 安全沙箱网关：笛卡尔积拦截、明细 500 行上限、Explain 行数熔断。

● 空结果自动诊断：筛选条件诊断与修正重试，业务 SQL 自动重试 3 次。

● 结构化意图帧：增强 Schema 检索；前端 sql_plan 结构化查询计划卡片。

04 📊 黄金 SQL 报表暂存

● 支持黄金 SQL 报表暂存、免模型极速直连运行；直连后「手动分析」快捷按钮与数据结果缓存复用。

05 🖼️ EmbedChat Canvas 多媒体画布

● Mermaid 架构图平移缩放；HTML 预览 / 代码双态 Tab。

● PDF / CSV / 图片超链接与附件区 Canvas 预览；物理路径自愈代理防裂图。

06 👤 用户画像、LTM 与澄清交互

● 全轮次注入只读用户画像与 LTM；真实姓名称谓与偏好提醒气泡。

● 澄清模块按场景路由，代码组装原因与 quick 按钮；修复澄清正文未输出 Bug。

07 ⚙️ ChatBI 架构重构

● DataAgentRunner 拆分为 26 个 chatbi/ 域模块；AgentService 重构；Pydantic V2 迁移；取消启动时 Redis 向量全量同步。

08 🎯 智能路由与体验优化

● 会话粘性支持极短追问（如「姓名呢」）；主助手工具预检；手动终止释放会话锁；元数据 DDL 导入优化。

⚠️ Breaking Changes & Migration Notes

从 v1.0.3 升级至 v1.0.4 时，请特别注意以下变更：

项目 说明
门户指令 主指令为 /dataset_portal，旧 /dataset_menu 仍兼容。
数据库变更 须执行 V79 / V80 / V81 三个增量脚本。
SQL 沙箱 笛卡尔积、超大明细等将被拦截；已移除自动注入 LIMIT。
查数超时 SQL 执行超时由 30s 调整为 60s。

🗄️ Database Incremental Upgrades

从 v1.0.3 升级至 v1.0.4 期间，平台数据库共引入了 3 个增量 SQL 升级脚本：

脚本文件 核心变更内容
V79-expand_meta_changelog_resource_id.sql 扩展 meta_changelog.resource_id 至 270 字符。
V80-update_data_api_timeout_seconds.sql 查数超时 30s → 60s。
V81-add_span_columns_to_traces.sql Trace 表新增 span_id / parent_span_id / meta_info。

📦 Upgrade Guide

从 v1.0.3 升级

# 1. 拉取最新代码

git fetch origin && git checkout main && git pull origin main

# 2. 更新依赖

source .venv/bin/activate

pip install -r requirements.txt

# 3. 执行数据库迁移（V79 / V80 / V81）

./db-prod/apply-sql-native.sh

# 4. 重新编译前端并启动

cd frontend && npm install && npm run build && cd ..

./dev.sh

完整部署说明请参考 HOW_TO_INSTALL.md。
