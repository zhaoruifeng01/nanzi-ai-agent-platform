Prompt 分层与 Trace

Studio · 轨迹字段 · 配额顺序 · 排障

项目声明

本项目为个人开源（MIT）。仓库：RandyChen1985/nanzi-ai-agent-platform

提示词分层 + Trace 观测，让「为什么这么答」可调试。配额在跑前拦截，Token 统计在中间件，历史落库后再喂下月配额。

Prompt Studio：/dashboard/prompts · API /api/portal/prompts · Trace GET /api/v1/chat/logs/{{trace_id}}

01｜Prompt Studio 能改什么

API

GET / 列表 · GET /detail?source&target_id&version · POST /test · POST /save（element:prompts:edit）· POST /optimize · GET /history
source：system_config | agent

层的现实

历史上的 SYSTEM_PROMPT_REGISTRY 已空；Router / Intent 的 DEFAULT_SYSTEM_PROMPT 仍在代码层（RouterService / IntentService）
Studio 主要编辑智能体系统提示（ai_agents / ai_agent_versions）。改完走版本发布，避免热改未发布草稿。

运行时还会叠：Skills 注入、LTM 画像、工具描述、用户消息——调试时对照 Trace 看实际进模型的轮次。

02｜Trace 记什么

AgentExecutionStep

step_number、event_type、agent_name、model、temperature、tool_name/input/output、raw_log、execution_time_ms、status、error_message、tokens、span_id/parent_span_id、meta_info、timestamp
event_type 常见：router、agent_execution、tool_call、thought、synthesis、few_shot、model_call
落库：AuditManager → AgentExecutionTrace + AgentExecutionHistory

03｜配额与统计顺序

三段式

1）跑前 QuotaService.check_before_call：按月汇总 AgentExecutionHistory.total_tokens；超限 yield quota_exceeded 并 return
解析顺序：admin → user → role → system
2）跑中 ModelCallStatsMiddleware → Redis LIST nanzi:{{uid}}:{{cid}}:model_call_stats
3）跑后 save_history 写 tokens，供下月配额
中间件只统计不写配额

04｜排障顺序

Checklist

1）版本是否发布、工具/Skills 是否勾选
2）Trace 看工具是否调用、参数是否合法、哪步报错
3）ChatBI 看 Gate 拒绝原因
4）Grounding 警告是否误伤预期
5）配额是否提前截断
6）第三方引擎则查对端日志（本地 Trace 较薄）

不要把生产密钥写进 Prompt；敏感配置走后台与环境变量。
