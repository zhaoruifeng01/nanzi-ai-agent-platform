上篇把架构与 AgentScope 运行时讲清楚了；这篇解决更土的问题：怎么把平台跑起来，以及二次开发该动哪里、不该动哪里。

NanZi 不是免运维 SaaS。你要自备 MySQL、Redis Stack、LLM 网关；知识库 / 部分链路还常要 RAGFlow。读完应能：选对部署路径、完成「能登录 + 配好模型」、知道升级怎么跑增量 SQL、扩展时优先走 MCP/Skills 而不是一上来改 Runner。

两条路径速记：生产 / 准生产优先 Docker 离线镜像包（可无本机装 Python/Node）；日常改代码优先 源码联调（venv + 前端构建）。无论哪条路，先初始化 MySQL，再拉服务。

01｜依赖地图：必装 / 按需 / 别装错

组件 级别 说明
MySQL 8+ 必装 utf8mb4；平台配置、RBAC、审计、智能体元数据、JobStore
Redis Stack 必装 必须带 RediSearch（如 redis/redis-stack-server）。普通 Redis 不够：LTM 向量、案例/元数据索引、调度锁、权限挂起、Token 会话账本都依赖它
LLM 网关 必配 OpenAI 兼容（OpenAI / DeepSeek / 百炼等）；登录后在【系统配置】填 Key 与 Base URL
Docker 20.10+ 生产必装 Compose v2；离线 docker load 部署
Python 3.11+ 源码必装 镜像基于 python:3.11-slim；AgentScope 模块（agent/model/tool/permission/workspace）需可 import。仅用现成镜像时可不上宿主机 Python
Node.js 18+ 源码/自构建 前端 Vue 构建；跨架构打镜像时脚本会在宿主机预编译前端，避免 Qemu OOM
RAGFlow 按需 知识库管理 / Knowledge 专家 / RAGFlow 引擎常见依赖；ChatBI 若走外部知识增强也常要。仅测通用对话可后配
业务库 ChatBI 按需 Oracle / ClickHouse / MySQL / SQL Server 等，在【数据源管理】登记

最常见翻车：① Redis 装了社区版却没有 RediSearch；② Docker 里把 MYSQL_HOST=localhost（容器里的 localhost 是自己）；③ Mac 打 x86 包却用了 build_native.sh。下文 FAQ 会再钉一次。

02｜无论哪条路：先初始化 MySQL

平台库脚本在 db-prod/，版本化文件名 V0…VNN-*.sql。心智模型：结构靠增量脚本演进，不是每次 drop 重建。

# 1) 建库

CREATE DATABASE nanzi_ai_agent_platform CHARACTER SET utf8mb4 …;

# 2) 导入结构（二选一，幂等可重复跑）

./db-prod/apply-sql.sh         # 需 venv + requirements

./db-prod/apply-sql-native.sh # 仅需 mysql 客户端

# 3) 首次建议导入管理员（脚本末尾也可级联询问）

./db-prod/apply-sql-native.sh db-prod/INIT-USER-ADMIN.sql

升级时怎么做？

拉新代码 / 换新镜像之后，对已有库再跑一次 apply-sql-native.sh（或 Python 版）。脚本会跳过已存在对象，把新的 V* 增量补上。发版说明（如 v1.0.7 Release Notes）会列出本版本新增脚本区间，升级前对照即可。

03｜路径 A：Docker 生产 / 离线部署

适合机房、内网、不想在服务器装 Node/Python 的场景。镜像把前后端打进同一服务；MySQL / Redis 一般外挂（你已有的实例）。

① 拿镜像：下载或自构建

· 推荐：GitHub Releases 下载对应版本的 linux-amd64 / linux-arm64 tar。

· 自构建（二次开发改过代码后）：

cd docker
./build_linux_x86.sh 1.0.7  # 常见 x86 服务器
./build_linux_arm.sh 1.0.7  # 鲲鹏 / Ampere
./build_native.sh 1.0.7    # 仅本机试跑

产物在 docker/release/。版本号参数必传。Apple Silicon 打给 x86 服务器时，务必用 build_linux_x86.sh，不要用 native。buildx 坏了（Homebrew docker + Colima 常见）先跑 ./install-buildx.sh。

② 目标机：load → 配 .env → 启动

docker load -i nanzi-ai-agent_…_linux-amd64_….tar
# 若 compose 写的是 :latest
docker tag nanzi-ai-agent:1.0.7 nanzi-ai-agent:latest

cd docker && cp ../env.example .env  # 再改 Host/密码/ENCRYPTION_KEY
./start-nanzi-ai-agent.sh
./stop-nanzi-ai-agent.sh        # 停止

.env 要点：MYSQL_HOST / REDIS_HOST 写宿主机局域网 IP 或 host.docker.internal（Mac/Windows Docker Desktop），严禁容器场景写 127.0.0.1。ENCRYPTION_KEY 必须配置（Fernet 类密钥）。接低版本 Oracle 时可能要 Thick 模式并挂载 Instant Client 卷——详见 docker/README.md。

Compose 有两套思路：docker-compose.ai-agent.yml 只起 API、外挂 DB/Redis（常见）；另有全栈编排可一并起依赖。选哪种以你环境为准，原则是平台库与 Redis Stack 地址对、健康检查能过。

04｜路径 B：源码开发联调

适合改 Runner、工具、前端、调 Trace。MySQL / Redis Stack 仍建议本机或局域网常驻服务，不要用假依赖糊弄向量检索。

python3 -m venv .venv && source .venv/bin/activate

pip install -r requirements.txt  # 含 agentscope[service,storage,workspace]

cp env.example .env              # 改 MySQL/Redis/ENCRYPTION_KEY

# 先完成 db-prod 初始化（见上一节）

# 日常联调：在项目根目录执行 ./dev.sh

# （停旧进程 → 快速编前端 → uvicorn --reload :8001）

# 或分步：uvicorn app.main:app --reload --port 8001
#         cd frontend && npm i && npm run dev

脚本怎么选？

· ./dev.sh：前台联调，跳过 vue-tsc，带 reload——日常改代码

· scripts/redeploy-fast.sh：后台快速热更新

· scripts/redeploy.sh：完整类型检查后后台发布，偏准生产

编译与启停请在你自己的终端执行；配好依赖后用上述脚本即可。

05｜能访问之后：登录与首次配置

服务起来后：管理台一般是 http://localhost:8001/，Swagger 在 /docs。

若导入了 INIT-USER-ADMIN.sql：默认用户 admin，用脚本中的预置 API Key 登录。后台初始常为「仅 API Key」模式——首次登录后立刻在用户管理/个人中心设登录密码，不要把预置 Key 长期挂在公网。

系统配置里必须先做的三件事

1. 大模型：厂商、API Key、API Base、默认模型 ID（路由/专家/合成都吃这里）

2. RAGFlow（要用知识库时）：API URL + Token；Docker 下同样别写容器内 localhost

3. 数据源（要用 ChatBI 时）：加连接、测通，再做元数据/授权（下篇建智能体细讲）

只装了平台、没配模型：页面能开，对话必挂。只配了模型、没 Redis Stack：登录也许还行，记忆检索 / 案例召回 / 部分调度会静默残废——优先查向量健康与 RediSearch。

06｜二次开发：该改哪里，不该改哪里

上篇说过：本地 AI 唯一运行时是 AgentScope；业务应尽量停在「配置与适配」，少硬耦上游对象。扩展按成本从低到高：

A. 不改核心代码（优先）

· MCP：登记 Server → 同步工具 → 智能体版本勾选（连 Jira / 内部 API 网关等）

· Generic API Tool：控制台配置 HTTP 工具 Schema，运行时进 Toolkit

· Skills：平台 / 个人 SKILL.md 包 + 版本白名单

· Prompt 工厂：architech/prompts/ 版本化系统提示词

· 场景模板 / 斜杠指令：交付包与门户入口，少写胶水代码

· Embed / V1 API：业务系统嵌入，不 fork 聊天页（第 11 篇）

B. 平台内扩展（要懂契约）

· 新静态工具：app/services/ai/tools/ 用平台 @tool 注册进 Registry → RuntimeToolSpec，不要直接绑 LangChain

· 改路由/意图：router_service / intent_service，先读设计文档再动

· 新Runner / 引擎路径：遵守 docs/md/ai_agent_gating_contract.md（工具权限、挂起恢复、SQL/真实性门控哪些必须）

C. 尽量别这么干

· 在业务代码里直接 import AgentScope 具体类绕过 runtime/agentscope 适配层

· 重新引入 LangChain 执行器 / Message / Tool（平台已明确迁出）

· 为图省事关掉 SQL Gate / 权限审批 / 工作区隔离「先跑通再说」——生产会反过来咬你

仓库目录速查（二次开发导航）

app/ FastAPI + AI 编排；重点 services/ai/

frontend/ Vue 管理台与 EmbedChat

db-prod/ 库表增量

docker/ 镜像与一键启停

architech/ 设计文档与 Prompt

tests/CHECKLIST.md 改功能记得勾验收

07｜FAQ：踩坑清单

Q：容器起来又退出，日志 Database health check failed？

A：MYSQL_HOST/REDIS_HOST 写成了 localhost。改成宿主机 IP 或 host.docker.internal。

Q：跨架构 build 前端 Killed / OOM？

A：构建脚本已支持宿主机预编译 Vite；确保本机有 node/npm。硬要在容器里编就加大 Docker 内存（≥8GB）。

Q：apply-sql 报 No module named aiomysql？

A：先激活 venv 并 pip install -r requirements.txt；或改用 apply-sql-native.sh。

Q：记忆搜不到、案例不召回？

A：先确认是 Redis Stack，且健康检查过 RediSearch；再查 REDIS_DB 等配置是否与文档一致。

Q：知识库/ChatBI 相关功能全挂？

A：检查 RAGFlow URL/Key、网络互通；ChatBI 另查数据源连通与元数据授权。通用助手可先不依赖它们验证编排链路。
