Skills 实战

双轨目录 · frontmatter · 白名单 · 提及/扫描/工具

项目声明

本项目为个人开源（MIT）。仓库：RandyChen1985/nanzi-ai-agent-platform

Skill = 可版本管理的「打法说明书」；系统提示词是人格；工具是手脚。三者叠用：提示词定角色，Skill 定流程与约束，工具执行。本篇按目录、字段、白名单、触发链路与对话工具拆开。

代码入口：skill_resolver.py · skill_metadata.py · system_executive_tools.py · API /api/portal/skills 与 /skills/personal

01｜双轨目录：平台 vs 个人

平台 Skills

干什么：落在 SKILLS_DIR：优先 /app/data/skills，否则 ~/.agents/skills。每个技能一目录 + SKILL.md。

场景：公司统一 SOP、问数规范、客服话术——管理员维护，全员可用。

原理：管理端 CRUD / toggle / zip 导入；需 menu:skills_management。scope=global。

个人 Skills

干什么：路径 agent_workspaces/{{user_key}}/skills/{{skill_id}}/SKILL.md。

场景：个人习惯、项目专用打法；不污染全局。

原理：API 前缀 /api/portal/skills/personal，登录即可，不强制 skills_management 菜单。同 id 时个人覆盖平台。

02｜SKILL.md frontmatter 字段

解析器保留的键

读文件头 8192 字节内 YAML：任意 key: value → 小写键。常用：id（默认目录名）、name、description、enabled（默认 true）。门户列表另支持 YAML 块标量。工具侧 create 至少写 name/description。
正文读取有路径 jail：必须在 SKILLS_DIR 或个人 skills 根下。

禁用：frontmatter enabled: false 或门户 toggle——加载与扫描都会跳过。

03｜版本白名单 skills_custom

AIAgentVersion 字段

skills_custom 默认 False：不限制公共技能（前端常显示「全部」，skills 存 []）
True：必须非空 skills JSON 字符串数组，只加载勾选的公共 id；个人技能仍合并
校验：custom=true 且 skills 空 → 保存失败

安全场景：生产主助手开 skills_custom，只挂「问数规范」「告警解读」等白名单，避免模型扫到实验技能。

04｜运行时三条触发路径

resolve_skills_from_query · 提及命中

干什么：用户话里「使用 XX 技能」等模式，或含「技能」/skill。

场景：显式点名：「用对账技能跑一遍上周差异」。

原理：匹配分：精确名/id=1.0，子串名=0.9，id=0.85，描述=0.7；阈值 ≥0.7；display name 子串兜底 0.75。默认 max_results=2，match_source=mention。

scan_relevant_skills · 自动扫描

干什么：未提及注入时，主助手可自动扫相关技能。

场景：用户没说「技能」但问题像某套 SOP。

原理：仅主助手：id sys-agent-chat 或 slug assistant|main|general-chat。配置：skill_auto_scan_enabled 默认 true；skill_auto_scan_min_score 默认 0.45；max_results 默认 1（钳制 1–3）。查询过短/寒暄不扫。个人技能 +0.05 分。

全文预加载阈值

干什么：高相关时把 SKILL.md 正文预注入，减少再调 read。

场景：命中后直接按说明书执行。

原理：skill_auto_full_load_enabled 默认 true；scan 需分 ≥ skill_auto_full_load_min_score（默认 0.75）；max_count 默认 1；max_bytes 默认 65536（上限 262144）。mention/mounted 可直接满载。

05｜对话工具三件套

list_available_skills

干什么：列出当前可见技能 id/name/description/scope。

场景：模型先盘点再选。

原理：尊重 skills_custom 与 disabled。

read_skill_instruction(skill_id)

干什么：读取 SKILL.md（≤262144 字节）。

场景：按说明书逐步执行。

原理：路径 jail + 可见性过滤。

create_skills(...)

干什么：对话里创建技能文件。

场景：沉淀本次打法为可复用 Skill。

原理：参数：skill_id/name/description/skill_md_content，scope 默认 personal（global 需 admin）。skill_id 正则 ^[a-zA-Z0-9_-]+$。成功回标记 NANZI_SKILL_CREATED:{{json}}。

边界：Skill 不能替代 RBAC、SQL Gate、工作区隔离、工具审批。它约束「怎么做」，不扩大「能碰什么」。
