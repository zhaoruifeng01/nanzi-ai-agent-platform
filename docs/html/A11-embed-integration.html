项目声明

本项目为个人开源（MIT）。仓库：RandyChen1985/nanzi-ai-agent-platform

业务系统不必重建聊天：Embed IFrame + postMessage，或服务端调 V1。鉴权与路由同一套。文档：docs/md/embed_integration_guide.md · api_integration_guide.md · chatbi_v1_http_api.md。

Widget：/embed/chat（EmbedChat.vue）；门户宿主 Chat.vue iframe 同地址。source 固定 nanzi-agent-embed。

01｜Host → Widget 事件

类型一览

INIT_CONFIG · OPEN_SAVED_REPORT · SYNC_STATE · UPDATE_CONTEXT · SET_THEME · STOP_GENERATION · CLEAR_SESSION · RESET_SESSION（可带 new_token）· SEND_COMMAND
可用 instance_id 过滤，防多实例串扰。

02｜Widget → Host 事件

已实现发射

NANZI_WIDGET_READY（挂载）· INIT_SUCCESS（token 有效）· GENERATION_STOPPED · CONVERSATION_CHANGED（含 clear_host_conversation_pin）· USER_FEEDBACK（message_id/trace_id/feedback）· OPEN_DATA_PORTAL_FULL（宿主可跳 /dashboard/personal?tab=data）

文档里的 REQUEST_RESIZE / TOKEN_EXPIRED 等：Debugger 可能监听，但当前 EmbedChat 未必发射——对接以代码 switch 为准。

03｜INIT_CONFIG 字段手册

鉴权与会话

干什么：注入 token 并定位智能体/会话。

场景：宿主登录后打开工位。

原理：token | api_key | apikey（必需其一）；agent_id；conversation_id（销钉 yovole_embed_conv_id）；instance_id；theme/styleVars；welcome_message_override；user_avatar；user_info|user；page_info；open_saved_report{report_id,run_id?,request_id?,detail_tab}；portal_question{query, action fill|send}。生产优先 postMessage 传 token，少写 URL。

水印

干什么：防截屏外泄。

场景：嵌入大屏/客服座席。

原理：用户 profile.watermark：enabled（配置 embedchat_watermark_enabled，缺省当 true）、style 默认 user_time、text 默认「南孜系统」。custom 用 text，否则用户名+时间瓦片，透明度约 0.04。

04｜V1 面要点（/api/v1）

对话

POST /chat/completions 自动路由或带 agent_id
POST /chat/agents/{{id}}/chat 直达
GET /chat/logs/{{trace_id}} 全步骤
history / cancel / upload / conversation finalize（强制 LTM flush）/ model_calls

ChatBI / 其他

POST /chatbi/sql/execute（需 sessionid + API Key）
POST /chatbi/sql/checkauth（public，body username）
POST /schema · GET /users/profile?username= 服务端换 token（Embed 指南）

完整表见 docs/md/api_integration_guide.md。权限确认/外部执行 resume 等也在 V1 chat 下。
