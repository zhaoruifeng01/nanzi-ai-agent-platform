工作台与场景模板

模式 · 八模板 · 深链 · 落地收束

项目声明

本项目为个人开源（MIT）。仓库：RandyChen1985/nanzi-ai-agent-platform

工作台把「待办注意力 + 最近结果 + 收藏智能体 + 推荐场景」收成首页；场景模板把行业打法一键安装。它是一线入口，不替代管理后台，也不绕过数据权限与 SQL Gate。

API GET /api/portal/workbench/home · 页 /dashboard/workbench · 模板服务 scenario_template_service.py

01｜Workbench 三种模式

active / quiet / new_user

组装规则：attention 非空 → active；否则若有 attention/results/resume 任一 → quiet；否则 new_user
上限：attention 5 · latest_results 6 · resume 4 · favorite_agents 6 · recommended_scenarios 4
来源：notifications / tasks / reports / conversations / agents / scenarios → source_status ok|empty|error

02｜八个内置场景模板 ID

manifest

1. chatbi-business-analysis
2. knowledge-qa-assistant
3. ops-inspection-assistant
4. finance-expense-analysis
5. sales-customer-insight
6. support-ticket-analysis
7. hr-policy-qa
8. legal-contract-review
前端：/dashboard/scenario-templates、/:templateId、/install

模板预置提示、推荐智能体/工具与示例问法；运营可扩 manifest，不必改 runtime。

03｜动作与深链

item actions

open_task / open_task_log → /dashboard/tasks?task_id&run_id
open_digest / open_report → /dashboard/chat?dataset_portal=1&report_id&run_id（可加 delivery=1）
open_conversation → ?conversation_id=
open_agent → ?agent_id=
open_scenario → /dashboard/scenario-templates/{id}

Chat / Embed 实际消费

Chat.vue 经 INIT_CONFIG 转发：conversation_id、agent_id、open_saved_report（report_id/run_id/…）、portal_question。
Slash：/dataset_portal；V1 导航仍走 /dataset-menu*
Embed 事件 OPEN_DATA_PORTAL_FULL → 宿主可去 /dashboard/personal?tab=data

04｜落地建议（第一季收束）

推广顺序

1）装好平台与 Redis Stack（A02）
2）建 ChatBI/知识专家并过就绪门禁（A03–A04）
3）挂工具/MCP/Skills（A05–A07）
4）通知+调度跑通无人值守（A08–A09）
5）需要时接第三方引擎或 Embed（A10–A11）
6）打开记忆/Grounding/Trace 做运营可观测（B01–B03）
7）工作台只推 1～2 个高价值场景模板，比「万能主助手」更可控
