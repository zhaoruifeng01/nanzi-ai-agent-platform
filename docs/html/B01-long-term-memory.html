长期记忆怎么工作

Key · 配置 · 工具 · 遗忘曲线 · 整理

项目声明

本项目为个人开源（MIT）。仓库：RandyChen1985/nanzi-ai-agent-platform

长期记忆让主助手跨会话记住偏好与事实：短时历史在 Redis LIST，会话摘要走 RediSearch 向量，偏好 LTM 在独立 HASH；夜间整理作业合并碎片。部署前提是 Redis Stack（RediSearch），DB0。

探测：GET /api/portal/memory/redis-vector-test · 管理页 /dashboard/memory

01｜Key 与索引（先记清）

短时 / 会话

conversation:{{uid}}:{{cid}}:history LIST，TTL 默认 7 天（memory_history_ttl_days）
conversation:{{uid}}:active 当前会话
另有 last_data_result / data_result_stack_v1

摘要与 LTM

memory:summary:{{uid}}:{{cid}} HASH+向量
memory:summary:daily:{{uid}}:{{yyyy-mm-dd}} 日汇总
nanzi:agent:ltm:{{uid}} 偏好/事实
memory:debounce:{{uid}}:{{cid}} 摘要防抖

RediSearch

索引名固定 nanzi:idx:memory:session_summary，PREFIX memory:summary:。字段含 user_id TAG、conversation_id TAG、title/summary TEXT、last_active NUMERIC、embedding VECTOR、reference_count、summary_type…
KNN：(@user_id:{{uid}})=>[KNN top_k @embedding $vec AS score]，相似度 score=max(0,1-distance)

02｜配置默认值

memory_service_configs

memory_service_enabled / memory_summary_enabled 默认 true
embedding：text-embedding-3-small · 1536 维
summary TTL 30 天 · history 7 天
debounce：3 轮或 300 秒
search KNN top_k=5 · summarize_min_assistant_chars=30 · max_sessions=50
memory_base_half_life=7.0 · memory_consolidation_threshold=0.82

03｜写入与工具

update_user_preference / fetch / delete

干什么：读写偏好 LTM HASH。

场景：「以后默认查华东机房」。

原理：走 ltm_service；user 从 agent context 取，禁止跨用户。

memory_search

干什么：跨会话检索摘要/历史。

场景：「上周我们怎么定的口径」。

原理：scope=summary|history|both；limit 默认跟 top_k=5。无 ctx.user_id 拒绝。

会话摘要

干什么：对话冷却后写 summary。

场景：长会话压缩进索引。

原理：SessionSummaryService → upsert_summary；防抖；也可 POST /api/v1/chat/conversation/{{id}}/finalize 强制 flush。

04｜检索注入与遗忘曲线

注入策略

偏好 LTM：should_inject_ltm 恒 True，fetch 超时 0.2s，格式化进画像提示
跨会话 hint：非 DATA_QUERY / SKILL_EXECUTION / KNOWLEDGE 时注入
命中后 HINCRBY reference_count+1

Ebbinghaus 启发重排

S = half_life × (1 + log1p(reference_count))
R = exp(-t_days / S)
final_score = score × R（排序用 final；UI 常展示原始相关度 score）

03:00 整理作业

Job system_memory_consolidation；锁 lock:system_memory_consolidation:{{YYYYMMDDHHMM}} NX EX 3600
按用户聚类：余弦 ≥0.82 合并，LLM merge，新 cid consolidated_{uuid}，summary_type=consolidated，继承 reference_count，删碎片

记忆 ≠ 知识库：KB 管文档；LTM 管「关于这个用户的偏好与约定」。混用会噪声。隔离：所有 KNN 带 @user_id。
