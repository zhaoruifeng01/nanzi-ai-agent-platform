Grounding：答案从哪来

opt-in · 七类证据 · 审计动作 · MCP

项目声明

本项目为个人开源（MIT）。仓库：RandyChen1985/nanzi-ai-agent-platform

Grounding = 答案是否「有据可查」。工具成功写入 EvidenceLedger，回合结束 GroundingService.audit；默认 opt-in（debug_options.grounding_enabled / 前端 enableGrounding 默认 false）。

代码：app/services/ai/grounding/ · EvidenceType / GroundingAction / EvidenceLedger

01｜开关位置

如何打开

请求体 debug_options.grounding_enabled=true
EmbedChat / AgentDebug：enableGrounding 默认 false，发送时映射；RESET_SESSION 会强制关回 false
执行器：_grounding_enabled() 仅当 is True

02｜EvidenceType 七类

枚举值

internal_data 问数/SQL
internal_knowledge 知识库
public_web 网页
runtime_state 运行时状态
user_file 用户文件
conversation_memory 记忆
external_tool MCP/外部工具

03｜策略与动作

记账策略

工具成功 → EvidenceLedger.record_success → EvidenceReceipt（类型、摘要、标记）
策略：non_empty / allow_empty_success / structured_success（空成功是否记账）

GroundingAction

pass · pass_with_warning · pass_without_facts · block_ungrounded_facts
风险级：none|low|medium|high。软警告路径：SSE log(category=grounding) + warning_chunk，附 grounding_risk{level,reason,required/available types}——PASS_WITH_WARNING 通常不删答案正文

04｜MCP 只读与证据

启发式

schema 可声明 x-nanzi-evidence-types
readOnlyHint=False → evidence_inference_disabled
True 且无声明 → EXTERNAL_TOOL + allow_empty_success
另有 Registry 名称动词启发式（见 A06）

边界：Grounding 证明「模型看过什么」，不自动证明业务结论正确。审计与人工复核仍要。
