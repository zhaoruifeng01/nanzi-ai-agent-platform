任务调度

字段 · Job/锁 · 系统作业 · 隐式工具

项目声明

本项目为个人开源（MIT）。仓库：RandyChen1985/nanzi-ai-agent-platform

任务中心让智能体「按时自己跑」。调度器用 APScheduler AsyncIOScheduler + MySQL JobStore 表 ai_agent_scheduler_jobs，时区 Asia/Shanghai；执行用 Redis 锁防多实例重入。

代码：scheduler_service.py · 表 ai_agent_scheduled_tasks（V38）· task_manager_tools.py

01｜AgentScheduledTask 字段

核心列

name、user_id、agent_id、conversation_id（创建常用 task_conv_{uuid12}）、cron_expr、prompt、source(web|agent)、status(0停/1跑/2错)、config(JSON：notification_channels、task_metrics…)、run_count、last_run_id/at、next_run_at。
跑批会话：{prefix}_run_{uuid12}，会话 id 长度上限 50。

健康指标（config.task_metrics）

trigger_count / success_count / failure_count / skipped_count / consecutive_failures / health_status / last_* / last_alert_at

02｜Job id 与 Redis 锁

Job 命名

用户任务 task_{{id}}
报表订阅 saved_report_subscription_{{id}}
系统：system_audit_log_maintenance · system_memory_consolidation · system_knowledge_metrics_sync · system_third_party_user_sync
用户任务/订阅 misfire_grace_time=3600

锁键 SET NX

lock:task_exec:{{task_id}}:{{YYYYMMDDHHMM}} TTL 300（非手动跑）
consolidation 锁 TTL 3600 · knowledge_metrics 300 · third_party_user_sync 3600
报表订阅不加同类执行锁

03｜系统作业时刻（上海时区）

默认 Cron

审计日志维护 02:00（保留天数默认 audit_log_retention_days=90）
记忆整理 03:00
知识指标同步 03:30
第三方用户同步：UserSyncService — hourly/daily(02:00)/weekly(周一 02:00)/off

04｜对话工具（SYSTEM_IMPLICIT）

SYSTEM_IMPLICIT_TOOLS 含：get_current_time + 下列任务工具；Registry 隐式集合还会追加记忆/技能/网页与 send_portal_notification。

create_recurring_task

干什么：创建周期任务。

场景：「每天 9 点汇总昨日签单」。

原理：参数 name、cron、prompt、notification_channels。Prompt 会包 【自动化指令-任务ID】+ 通知补充。source=agent。

get_my_tasks / cancel / start / pause / run_task_manually

干什么：列表与启停、手动跑一次。

场景：对话里运维自己的 Cron。

原理：start→status=1；pause→0；manual 调 run_task(..., is_manual=True)，手动跑不抢时间片锁。

改 Cron 注意 Asia/Shanghai；Redis 抖动会同时影响调度锁与会话历史。
