NanZi 开源智能体平台实战连载 · 03

三种智能体怎么选？

类型 · 向导五步 · engine_config · 就绪门禁

本项目为个人开源（MIT）。GitHub

本篇按控制台真实流程讲：选型 → 版本向导五步 → 引擎配置字段 → 就绪门禁 missing 码 → 发布。不是「点几下就好了」的空教程。

01｜主类型与锁定能力

三类枚举

GENERAL→general_chat；CHATBI→data_query；KNOWLEDGE_BASE→knowledge_base。
normalize_agent_capabilities 保证主能力在首位，扩展标签可追加但不能替换锁定项。存库字段：AIAgent.agent_type + JSON capabilities。

引擎是另一轴：LOCAL / RAGFLOW / OPENCLAW。类型选错，工具挂再全也别扭。

02｜版本配置向导（VersionConfigStep）

五步

① agent 智能体信息 · ② model 模型策略 · ③ tools 工具能力 · ④ prompt 系统提示词 · ⑤ review 确认保存
新建 LOCAL：五步全走。编辑已有 LOCAL：从 model 起（不再改 agent 步）。新建 RAGFLOW/OPENCLAW：通常只走 agent（外部引擎不吃本地版本工具/提示词向导）。

发布状态机

草稿保存 → publishVersion → 服务端 evaluate_agent_readiness → 旧 PUBLISHED 归档为 ARCHIVED → 目标版本 PUBLISHED → onboarding COMPLETE。LOCAL 运行时只认已发布版本上的模型/工具/提示词/Skills。

03｜engine_config 字段（按引擎）

LOCAL

可选 dataset_ids（知识增强）、ragflow_similarity_threshold / ragflow_vector_weight（运行时可能归一进 rag_params）。主能力靠类型锁定；问数工具在版本 tools 里勾。

RAGFLOW（必填 app_id）

必填 app_id；可选 dataset_ids、相似度/向量权重。Dispatcher → RAGExecutor，打 RAGFlow OpenAI 兼容 Agent 接口。

OPENCLAW（必填 base_url + model）

必填 base_url、model（机器人 ID）；可选 api_key、safety_check_enabled（默认开）、input/output prompt 与 strategy（默认 append）、stream 等。→ OpenClawExecutor。

04｜就绪门禁 missing 码

evaluate_agent_readiness

· published_version — 没有已发布版本
· primary_capability — 缺锁定主能力
· data_query_tool — CHATBI 未启用 get_dataset_schema/execute_sql_query 任一
· knowledge_base_binding — KB 的 engine_config.dataset_ids 为空
· knowledge_base_tool — 未启用 search_knowledge_base
同一套规则约束发布与 sub_agent_call 委派。

05｜三类最小可用 checklist

GENERAL

发布版 + general_chat。实战再挂通知/办公/HTTP/任务；主助手开工具预检与 Skill 扫描。

CHATBI

发布版 + data_query + 问数工具 + 数据源/元数据/数据集授权。门户 /dataset_portal。护栏见第 04 篇。

KNOWLEDGE_BASE

发布版 + knowledge_base + dataset_ids + search_knowledge_base。菜单权≠全库可读。

十分钟验收：通用能触发已挂工具；ChatBI 简单计数 Trace 见 schema/SQL；知识库回答带引用。再叠 MCP/Skills。
