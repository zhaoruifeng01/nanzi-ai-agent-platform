项目声明

本项目为个人开源（MIT）。仓库：RandyChen1985/nanzi-ai-agent-platform

引擎类型 ≠ 智能体主类型。GENERAL/CHATBI/KNOWLEDGE_BASE 是能力画像；LOCAL / RAGFLOW / OPENCLAW 是运行时后端。RAGFLOW/OPENCLAW 早退，不走本地 Toolkit、SQL Gate、Skills 全套护栏。

代码：dispatcher.py · RAGExecutor / OpenClawExecutor · openclaw_client.py · sanitize_client_messages_for_identity

01｜分发路径

AgentDispatcher

engine_type==RAGFLOW → RAGExecutor → RagFlowClient → POST {{base}}/api/v1/agents_openai/{{app_id}}/chat/completions（timeout 60s）
OPENCLAW → OpenClawExecutor → POST {{base_url}}/v1/chat/completions（timeout 180s）
否则 LOCAL：按能力走 Knowledge / DataQuery / Assistant

创建 UI 常把第三方引擎固定为 GENERAL + general_chat；不走本地版本化工具清单那一套。

02｜engine_config 必填与默认

RAGFLOW

干什么：托管 RAGFlow Agent App。

场景：已有 RAGFlow 应用，想统一入口。

原理：必填 app_id；可选 dataset_ids、ragflow_similarity_threshold、ragflow_vector_weight。检索默认相似度 0.2、向量权重 0.3、top_k 5（可被系统配置 knowledge_ragflow_* 覆盖）。平台凭据：ragflow_api_url / ragflow_api_key。

OPENCLAW

干什么：走 OpenClaw 网关聊天。

场景：要数据集 ACL 与外部机器人编排。

原理：必填 base_url、model；可选 api_key、stream 默认 True、extra_params。安全：safety_check_enabled 默认 True；input/output strategy 默认 append；可配 safety_check_*_prompt。平台回落 openclaw_api_url/api_key。

03｜AUTH_CONTEXT（OpenClaw）

服务端注入 JSON 键

userName · realName · channel=openai-user · role=普通用户 · datasets 数组
数据集元素：ragflow_meta_id / ragflow_meta_name / ragflow_meta_desc（来自 PermissionService 可访问元数据集）
不含 apiKey。作为首条 system 插入；空 datasets 时 Prompt 要求拦截 ChatBI。请求 user 字段常为 {user}-{conversation_id}。

sanitize：丢弃客户端 system；剥离伪造的 AUTH_CONTEXT / USER_PROFILE / Active User Profile 等标记块。身份只信服务端。

04｜怎么选

决策

要平台护栏 / SQL Gate / Skills / 工具审批 → LOCAL
复用 RAGFlow 托管 App → RAGFLOW
走 OpenClaw 网关与数据集 ACL → OPENCLAW
外部能力受对端版本、网络与对方鉴权约束——平台只代理，不内化其工具权限模型。
