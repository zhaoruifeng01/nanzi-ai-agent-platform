消息通知怎么配

四通道 · 凭据字段 · send_* · 报表简报与节流

项目声明

本项目为个人开源（MIT）。仓库：RandyChen1985/nanzi-ai-agent-platform

通知通道是用户自己的凭据配置；任务/报表只选 channel id，不让模型在 Prompt 里钓 webhook。本篇覆盖四通道、个人中心字段、对话工具、黄金报表简报流水线与失败节流。

代码：task_notification_channels.py · notification_service.py · saved_report_digest_service.py · API /api/portal/notifications / /inbox

01｜四通道 CHANNEL_SPECS

channel_id → 工具

portal → send_portal_notification（站内铃铛）
dingtalk → send_dingtalk_message
wechat_work → send_wechat_work_message
email → send_email
配置键 notification_channels；任务中心 UI 默认常勾 ["portal"]。投递补充文案会要求「每个渠道只发送一次」。

02｜个人中心凭据字段

表 user_notification_configs

列：user_id、channel_type（仅 dingtalk|wechat_work|email）、config_json；唯一 (user_id, channel_type)。portal 不走此表，直接写 Inbox。

钉钉 JSON

干什么：存 webhook 与加签 secret。

场景：任务失败告警、报表简报。

原理：默认 is_enabled=False、webhook_url=""、secret=""。MASKED_KEYS 含 secret → API 回 ******。

企微 JSON

干什么：只存 webhook_url。

场景：群机器人简报。

原理：默认 is_enabled=False、webhook_url=""。

邮件 JSON

干什么：SMTP 发信。

场景：对领导的日报邮件。

原理：smtp_host/port(默认465)/user/password、sender_name 默认「AI Agent」。smtp_password 脱敏。

API

GET/PUT /api/portal/notifications/config · POST .../test 试发
Inbox：GET /inbox、unread-count、read、read-all、删除等

03｜对话里的 send_* 工具

send_portal_notification

干什么：写站内消息。

场景：任务跑完提醒本人。

原理：参数 title/content/level(info|success|warning|error)。category=agent，resource_type=agent_message。属于隐式工具集合（常自动可用）。

send_dingtalk_message / 企微 / 邮件

干什么：读用户配置发外部消息。

场景：把结论推到群或邮箱。

原理：钉钉 title+content（可选 HMAC secret）；企微 content；邮件 to_email+subject+content，SMTP 用个人配置。未启用或未配 URL 会失败提示。

反模式：在系统提示词里写死 webhook。正确做法是任务 config 选 channel id，运行时读 UserNotificationConfig。

04｜黄金报表简报流水线

触发

Job id saved_report_subscription_{{id}}，CronTrigger，时区默认 Asia/Shanghai，misfire_grace_time=3600。无 Redis 任务锁（与用户 Task 不同）。命中 notify_on_success + 告警条件 hit 后走消化。

三步生成

1）build_deterministic_digest：MAX_RECORDS=5、MAX_FIELDS=5、MAX_CONTENT_LENGTH=3800；mode=fallback
2）enrich_digest_with_ai（订阅 ai_analysis_enabled 默认 True）：源行上限 10，temperature=0.1；失败回退 fallback
3）深链 APP_PUBLIC_URL/dashboard/chat?dataset_portal=1&report_id=&run_id=
4）render_mobile_markdown：统计范围/核心结论/AI 分析/关注事项/关键数据/查看完整报表——不用 Markdown 表格

投递与节流

始终写 Inbox（resource_type=saved_report_run）+ 审计 channel=inbox；再按 external_channels 外发并各记 delivery。
失败：Inbox error 必达；外发仅当 notify_on_failure 且 consecutive_failures==1 或 %3==0（与任务中心告警同思路）。订阅默认 notify_on_success=False、notify_on_failure=True。
