# 编辑智能体紧凑布局设计

## 目标

在不改变保存语义的前提下，降低编辑智能体弹窗高度，提高字段扫描效率，并明确区分“不可修改的分类信息”和“可修改的引擎参数”。

## 布局

- 弹窗使用 `max-w-3xl`，内容区保持有限高度和内部滚动。
- System Agent 开关进入标题栏右侧，不再占用独立内容行；仅管理员可见。
- 物理标识符、显示名称和排序权重组成紧凑三列。
- 描述信息保持整行，默认两行高度。
- 智能体类型和高级能力设置采用紧凑摘要/折叠样式。
- 取消、保存操作进入固定底部栏。

## 引擎边界

- 编辑时执行引擎类型不可切换，只展示当前引擎徽标和“创建后不可修改”说明。
- LOCAL 不展示外部引擎参数。
- RAGFlow 仍可编辑 App ID、Dataset IDs、相似度阈值和向量检索权重。
- OpenClaw 仍可编辑地址、API Key、机器人 ID，以及输入/输出安全审查规则。
- OpenClaw 安全审查高级参数默认折叠；开关和当前状态保持可见。

## 兼容性

- `Modal` 新增可选标题栏附加区域和底部区域，不影响没有使用插槽的现有弹窗。
- 不修改 API、保存字段、智能体类型锁定规则和权限判断。
