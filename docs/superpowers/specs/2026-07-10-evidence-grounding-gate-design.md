# 通用事实取证门禁设计

## 目标

为 Main 和不具备相应事实能力的通用/自定义智能体增加平台级事实取证门禁。稳定公共知识、写作、翻译、代码与推理任务可直接回答；当前、私有、用户特定、文件特定、易变化或要求精确来源的事实，必须由本轮真实工具执行产生可信证据后才能输出。

## 原则

1. 提示词不是信任根，模型声称“已查询”不能构成证据。
2. 门禁依据抽象事实来源和工具能力，不依据销售、资产、业务员等具体业务词。
3. 工具成功事件由服务端签发证据；失败、超时、权限中断和空结果不签发有效证据。
4. 需要外部事实的回答先缓冲、后审核；非事实任务保持原有流式路径。
5. 分类不确定且候选回答包含外部事实声明时 fail closed，只允许方法说明或明确的未取证说明。

## 事实来源

- `internal_data`：数据库、数据集、业务 API、数据智能体。
- `internal_knowledge`：企业知识库、制度、SOP。
- `public_web`：新闻、官网和实时公开资料。
- `runtime_state`：服务器、进程、任务和服务状态。
- `user_file`：用户附件和工作区文件内容。
- `conversation_memory`：历史会话记忆和长期偏好。

## 组件

### FactRequirementResolver

复用统一请求决策，输出本轮是否需要证据、可接受证据类型和判定置信度。它只确定来源能力，不选择具体工具或智能体。`unknown` 不再等价于可无条件回答：若输出包含具体外部事实，则必须有证据。

### EvidenceLedger

Ledger 存放在本轮 AgentScope state 中。`RuntimeToolSpec` 声明可生产的证据类型和结果有效策略；只有真实 `success` 审计事件且结果满足策略时才登记 `EvidenceReceipt`。凭证绑定调用 ID、工具、用户、会话、来源类型和结果摘要。

### GroundedResponseGate

Main 的最终正文进入门禁。若请求需要外部事实或请求来源不确定，门禁检查本轮 Ledger。存在匹配证据则放行；没有证据且候选正文包含具体外部事实或虚假执行声明时，替换为服务端安全答复。稳定知识与非事实任务继续放行。

## 执行与降级

取证请求优先强制调用能生产相应证据的工具或子智能体。没有可用能力、权限拒绝、调用失败或空结果时，不生成具体事实，只提供失败原因、重试/切换入口或分析方法。

## 兼容性

第一期接入 Main 和通用 Assistant Runner；ChatBI、Knowledge Runner 保留现有领域门禁，同时可逐步登记统一凭证。旧 `_is_hallucinated_data_reply()` 暂留作兼容兜底。新门禁配置默认先以 Main 开启、其他无事实能力智能体开启，专业事实智能体由自己的执行闭环负责。

## 验收

- 无工具生成内部数据、知识库结论、联网事实、服务器状态或文件内容时被拦截。
- 声称调用工具但不存在服务端成功凭证时被拦截。
- 调用无关工具、失败工具或空结果后仍编造事实时被拦截。
- 有匹配证据时正常输出。
- 稳定常识、写作、代码、翻译、数学和显式假设示例不强制取证。
- 非事实任务不增加模型调用；取证任务除必要工具外不增加审核模型调用。
