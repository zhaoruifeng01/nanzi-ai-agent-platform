# ChatBI 监控配置弹层设计

## 目标

用产品内统一弹层替换 `window.confirm`，让用户在创建监控前明确配置执行频率、执行时间和成功通知。EmbedChat 与 AgentDebug 使用同一组件和同一请求契约。

## 交互

1. 用户在“继续分析”中点击“创建监控”。
2. 打开居中业务弹层，标题为“创建指标监控”，展示当前结果来源提示。
3. 默认值保持现有行为：每天、09:00、成功后通知。
4. 用户可选择：
   - 执行频率：每天、每周、每月；
   - 每周时选择星期；每月时选择日期；
   - 执行时间；
   - 是否在执行成功后通知。
5. 点击“创建监控”后按钮进入加载态并禁止重复提交。
6. 创建成功后关闭弹层并显示 Toast；失败时弹层保持打开，在表单底部显示服务端错误。
7. 点击取消、关闭按钮或遮罩关闭弹层；提交期间不可关闭。

## 组件边界

新增共享组件 `frontend/src/components/chatbi/ChatBIMonitorDialog.vue`：

- 输入：`open`、`resultId`、`conversationId`；
- 输出：`close`、`created`；
- 组件负责表单、校验、请求和提交状态；
- 两个聊天页面只负责保存待创建监控的 action、打开弹层和显示成功 Toast。

不在两个大型页面中复制弹层模板或请求逻辑。

## API 契约

继续调用 `POST /api/portal/chatbi-monitors`：

```json
{
  "conversation_id": "conversation-id",
  "result_id": "result-id",
  "schedule_type": "daily",
  "time_value": "09:00",
  "weekday": null,
  "monthday": null,
  "notify_on_success": true
}
```

每周和每月分别填写 `weekday`、`monthday`。后端现有幂等逻辑继续防止相同结果重复创建。

## 样式与可访问性

- 延续现有 Tailwind 弹层：半透明遮罩、白色/深色卡片、圆角和阴影；
- 层级高于“继续分析”菜单；
- 使用 `role="dialog"`、`aria-modal="true"` 和明确标题；
- 桌面端为紧凑居中弹层，移动端保持安全边距并允许内容滚动；
- 不再出现“localhost 显示”等浏览器原生文案。

## 校验与错误处理

- 时间必填且格式为 `HH:mm`；
- 每周必须选择星期；
- 每月日期限制为 1–28，避免月份长度差异；
- API 错误优先展示 `response.data.detail`，否则展示通用错误；
- 相同结果再次创建时，后端返回 `created: false`，前端提示“该结果的监控已存在”。

## 测试

- 前端契约测试：两个页面不再包含 `window.confirm`，均挂载共享组件；
- 组件测试/契约：默认值、频率字段切换、请求参数、加载态和错误态；
- 后端现有监控幂等与字段校验测试继续执行；
- 人工验证桌面端、移动端和深色模式。

## 非目标

- 本次不实现复杂 Cron 编辑器；
- 本次不在创建弹层中配置阈值告警，告警条件仍在黄金报表订阅设置中维护；
- 本次不改变监控调度器和通知渠道实现。
