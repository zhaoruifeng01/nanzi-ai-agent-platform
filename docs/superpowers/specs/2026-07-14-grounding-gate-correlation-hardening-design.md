# 事实取证门禁相关性与全链路一致性优化设计

## 1. 背景与目标

现有门禁已经把 Main 的事实取证失败从阻断改为正文内风险提示，也已经为只读 MCP、原生工具、知识库、文件和 ChatBI 数据结果登记类型化凭证。本轮优化解决审查中确认的两类问题：一是无关、失败或空结果凭证可能错误放行具体事实；二是 KnowledgeAgent、ChatBI、外部执行恢复和 MCP 结构化返回没有完全遵循同一套软门禁契约。

目标是：正确工具结果尽量不误报，无关或失败工具结果不能为回答背书；所有事实型 Runner 最终都保留正文并使用消息内风险提示，不重新引入事实阻断卡片。权限、SQL、路径、写操作确认等安全门禁继续硬阻断。

## 2. 核心设计

### 2.1 统一成功结果规范化

`EvidenceLedger` 对字符串、字典和对象采用一致的失败判断。以下结果不得签发凭证：`isError/is_error=true`、`success=false`、`state/status` 为失败、HTTP/RPC 错误、权限拒绝以及非空 `error` 字段。判断覆盖顶层标准 envelope，但不把正文中讨论“错误排查”的正常知识内容误判为工具失败。

MCP 同时提取文本 `content` 与 `structuredContent`。只有结构化内容时仍作为真实结果返回；空文本且无结构化内容时才返回显式空成功 envelope。

### 2.2 事实相关性从单词命中升级为强关联

凭证继续保存不可逆哈希标记，不落明文。标记区分强标记与弱标记：车次、订单号、日期时间、金额、百分比、IP、较长英文/数字标识属于强标记；常见地点、二字中文词和通用字段名属于弱标记。

非空结果支撑事实回答时满足以下任一条件：

1. 至少一个强标记匹配；
2. 至少两个不同的普通标记匹配；
3. 回答不包含具体事实，只是在复述显式无结果状态。

精确证据类型不再无条件通过。候选回答包含动态事实、数字或结构化事实时，精确类型也必须通过内容关联；不含外部事实信号的说明性回答仍直接通过。

### 2.3 空结果只支撑“无结果”

空成功凭证只在回答明确表达未找到、暂无、无法取得结果，并且回答不含具体金额、数量、百分比、日期时间、标识符或事实表格时通过。出现“暂无，但 G1 还有 10 张”一类混合表述时必须追加高风险提示。

### 2.4 Runner 行为统一

- Main：继续保留正文并追加一次风险提示。
- KnowledgeAgent：保留最多两轮反思修正；最终仍未通过时输出最后一次回答或克制的无法核实说明，并追加消息内风险提示，不再发送“最终拦截”错误状态。
- ChatBI：不改变 Schema、SQL 预检、权限、修复和联邦查询流程；在有成功数据结果的最终综合阶段，对最终文本与当前/复用数据结果做相关性检查。缺少成功数据结果却输出具体业务事实时追加风险提示。SQL 安全失败仍按原逻辑终止。
- 外部执行恢复：只为状态为 success、且能匹配挂起工具定义的结果登记对应证据；错误、拒绝和未知工具结果不登记。

### 2.5 请求分类兜底

Router 的来源判定继续作为主要触发依据。对被分类为 GENERAL 但回答包含高置信动态事实信号的情况，执行轻量 UNKNOWN 输出审查；普通知识、数学、代码、创作、翻译和显式假设不触发。

## 3. 前端与兼容性

后端不产生新的 `grounding_blocked` 事件。现有前端阻断卡片解析本轮不删除，以免影响历史会话或旧后端兼容，但增加回归测试确保新 Main、Knowledge 和 ChatBI 路径不发送该事件。风险提示仍是普通 Markdown 内容，并附带 `grounding_risk` 元数据。

## 4. 测试与验收

必须覆盖以下失败优先用例：

1. `{"isError": true}` 和 `{"state": "error"}` 不签发凭证。
2. 车票结果中的“上海”不能支撑上海天气结论。
3. 无关 `public_web` 或 `internal_data` 回执不能支撑另一组具体事实。
4. 空结果加编造车次/金额必须提示。
5. 正确车次、时间、价格结果与回答匹配时不提示。
6. MCP 只有 `structuredContent` 时保留结构化数据并签发凭证。
7. KnowledgeAgent 最终失败不阻断正文。
8. 外部执行成功结果登记凭证，错误结果不登记。
9. ChatBI 最终回答超出 SQL 结果时提示，匹配时不提示。
10. 普通知识、代码、数学、创作和假设示例不新增提示。

## 5. 非目标

本轮不实现完整自然语言蕴含模型，不要求逐字引用，也不重构 Router、SQL 执行器或前端消息架构。相关性采用可解释的标记规则与保守阈值，后续通过风险提示率和人工反馈迭代。
